# Cursor @ 記号活用ガイド

Cursor で@記号を使用すると、AI に様々なコンテキストを提供でき、より正確で効果的なコード生成が可能になります。このガイドでは、各@機能の使用法と活用シナリオを詳しく説明します。

## 📋 @ 記号概要

@記号は、Cursor で AI に特定のコンテキストを参照させる強力なツールです。矢印キーで提案を探索し、`Enter`で選択できます。カテゴリ（例：Files）を選択すると、そのカテゴリ内の関連項目がフィルタリングされて表示されます。

## 🔧 主要な @ 機能

### 1. **@Files** - ファイル参照

#### 使用法

```
@filename.js
@package.json
@README.md
```

#### いつ使用するか

- AI が特定のファイル内容を参照する必要がある時
- ファイルの構造や設定に基づいてコードを生成する時
- 既存のファイルとの一貫性を保ちながら開発する時

#### 活用例

```
@package.jsonを参照して新しい依存関係を追加するnpm installコマンドを作成してください
@config.jsの設定に基づいてデータベース接続コードを書いてください
```

#### 利点

- ファイル自動補完機能による高速選択
- 正確なファイル内容に基づくコード生成

#### 欠点

- 大きなファイルの場合、不要なコンテキストまで含まれる可能性

---

### 2. **@Folders** - フォルダ参照

#### 使用法

```
@utils/
@components/
@src/services/
```

#### いつ使用するか

- AI がフォルダ全体の構造を理解する必要がある時
- フォルダ内のほとんどのファイルが関連する作業を行う時
- フォルダのパターンに合わせて新しいファイルを作成する時

#### 活用例

```
@components/フォルダの構造を参照して新しいButtonコンポーネントを作成してください
@utils/フォルダのパターンに従って新しいユーティリティ関数を書いてください
```

#### 利点

- フォルダ全体のコンテキストを一度に提供
- 一貫したコードスタイルと構造の維持

#### 欠点

- 大きなフォルダの場合、過度なコンテキストによる性能低下の可能性

---

### 3. **@Code** - コードシンボル参照

#### 使用法

```
@functionName
@ClassName
@CONSTANT_NAME
@variableName
```

#### いつ使用するか

- 特定の関数やクラスの実装を参照する時
- 既存のコードパターンに従って新しいコードを書く時
- リファクタリングや拡張作業を行う時

#### 活用例

```
@LRUCachedFunction実装を参照して似たようなキャッシング関数を作成してください
@UserModelクラスに基づいてAdminModelクラスを書いてください
```

#### 利点

- 非常に正確なコード参照
- 特定のシンボルに対する正確なコンテキスト提供

#### 欠点

- コードベースの深い理解が必要
- 正確なシンボル名を知る必要がある

---

- 最新の API 使用法の反映

#### 欠点

- サポートされるライブラリが限定的である可能性

## 🎯 効果的な @ 記号活用戦略

### 1. **コンテキストの組み合わせ**

```
@package.json @src/config/ @Web "Node.js環境設定ベストプラクティス"を参照して
本番環境設定を作成してください
```

### 2. **段階的アプローチ**

1. まず`@Files`で関連ファイルを確認
2. `@Code`で特定の関数やクラスを参照
3. `@Docs`で公式ドキュメントを確認
4. `@Web`で最新情報を補完

### 3. **プロジェクト段階別活用**

- **初期設定**: `@package.json`, `@Docs`, `@Web`
- **開発中**: `@Code`, `@Files`, `@Cursor Rules`
- **デバッグ**: `@Linter Errors`, `@Recent Changes`, `@Git`
- **リファクタリング**: `@Past Chats`, `@Code`, `@Folders`

## 💡 活用のコツ

### 1. **具体的なコンテキストの提供**

❌ 悪い例: `@utils`
✅ 良い例: `@utils/dateHelper.jsのformatDate関数を参照して`

### 2. **必要な分だけ参照**

- 大きなファイルよりも特定の関数やクラスを参照
- フォルダ全体よりも関連ファイルのみを選択

### 3. **組み合わせの活用**

```
@components/Button.tsx @Cursor Rulesを参照して
新しいIconButtonコンポーネントを作成してください
```

### 4. **リアルタイム情報の活用**

```
@Web "React 18 Suspense使用法"を参照して
ローディング状態を処理するコンポーネントを作成してください
```

```
ユーザー認証システムを実装してください
```

### 2. **コードレビューと改善**

```
@Recent Changes @Linter Errors @Cursor Rulesを参照して
最近の変更をコードスタイルに合わせて改善してください
```

### 3. **ドキュメント化とテスト**

```
@src/api/ @Past Chatsを参照して
APIエンドポイントのテストコードとドキュメントを作成してください
```

## 📝 注意事項

1. **コンテキストサイズ管理**: 過度なコンテキストは性能低下を引き起こす可能性があります。
2. **関連性確認**: 参照するコンテキストが実際に作業と関連があるか確認してください。
3. **最新情報の活用**: `@Web`を活用して最新の技術動向を反映してください。
4. **チームルールの遵守**: `@Cursor Rules`を活用して一貫したコードスタイルを維持してください。

## 🎉 まとめ

@記号は、Cursor の AI 機能を最大限活用できる強力なツールです。各状況に適した適切な@記号を選択し、組み合わせて使用することで、より正確で効率的なコード開発が可能になります。

最初は基本的な`@Files`と`@Code`から始めて、段階的に他の機能を習得することをお勧めします。
